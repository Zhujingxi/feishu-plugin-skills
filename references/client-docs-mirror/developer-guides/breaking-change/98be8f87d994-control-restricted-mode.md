# 获取消息能力补充群保密设置等场景管控

## 概述
为了进一步保障消息安全与用户隐私，并确保开放平台能力与客户端展示逻辑一致，我们将对消息和群组相关 OpenAPI 接口和事件进行升级。

核心变更： 针对“群聊保密设置”、“系统消息可见性”及“消息禁止转发”等场景，开放平台将补充严格的权限管控。变更生效后，如果该消息不可见或不可复制，将无法通过OpenAPI接口或事件获取受限的消息内容或资源。

## 生效时间

本次不兼容变更预计于 2026 年 3 月 24 日起逐步生效，具体灰度与全量节奏将以开放平台实际发布为准。

## 变更详情

### OpenAPI

1. **[获取指定消息的内容](https://open.feishu.cn/document/uAjLw4CM/ukTMukTMukTM/reference/im-v1/message/get)：GET /open-apis/im/v1/messages/:message_id**
    - 访问开启保密模式禁止复制消息、访问禁止转发消息、访问不可见的系统消息，将收到230050错误码（该错误码历史已对开发者暴露）。

2. **[获取会话历史消息](https://open.feishu.cn/document/uAjLw4CM/ukTMukTMukTM/reference/im-v1/message/list)：GET /open.feishu.cn/open-apis/im/v1/messages**
    - 访问开启保密模式禁止复制消息、访问禁止转发消息、访问不可见的系统消息，接口不会报错，无权限获取的消息会被过滤。
3. **[获取消息中的资源文件](https://open.feishu.cn/document/uAjLw4CM/ukTMukTMukTM/reference/im-v1/message-resource/get)：GET /open.feishu.cn/open-apis/im/v1/messages/:message_id/resources/:file_key**
    - 访问开启保密模式禁止复制消息、访问禁止转发消息、将收到234038错误码（该错误码历史已对开发者暴露）。

### 开放事件

对于[新版本消息事件](https://open.feishu.cn/document/uAjLw4CM/ukTMukTMukTM/reference/im-v1/message/events/receive)（im.message.receive_v1）、 历史版本接收消息事件(message)：当应用收到群聊保密模式禁止复制消息管控时、系统消息不可见时，将不再推送消息事件。

## 开发者建议

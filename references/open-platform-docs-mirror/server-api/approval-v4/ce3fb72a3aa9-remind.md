# 发送催办消息

当需要催促审批人审批单据时，通过该接口给审批人发送催办消息

## 请求

基本 | &nbsp;
---|---
HTTP URL | https://open.feishu.cn/open-apis/approval/v4/instances/remind
HTTP Method | POST
接口频率限制 | [100 次/分钟](https://open.feishu.cn/document/ukTMukTMukTM/uUzN04SN3QjL1cDN)
支持的应用类型 | Custom App、Store App
权限要求<br>**调用该 API 所需的权限。开启其中任意一项权限即可调用** | 操作审批实例(approval:instance:write)

### 请求头

名称 | 类型 | 必填 | 描述
---|---|---|---
Authorization | string | 是 | `user_access_token`<br>**值格式**："Bearer `access_token`"<br>**示例值**："Bearer u-7f1bcd13fc57d46bac21793a18e560"<br>[了解更多：如何选择与获取 access token](https://open.feishu.cn/document/uAjLw4CM/ugTN1YjL4UTN24CO1UjN/trouble-shooting/how-to-choose-which-type-of-token-to-use)
Content-Type | string | 是 | **固定值**："application/json; charset=utf-8"

### 请求体

名称 | 类型 | 必填 | 描述
---|---|---|---
instance_code | string | 是 | 审批实例 Code，可通过「获取单个审批实例详情接口」得到，与其他系列接口中instance_code是同一概念<br>**示例值**："81D31358-93AF-92D6-7425-01A5D67C4E71"
task_ids | string\[\] | 是 | 被催办的任务ID，通过「获取单个审批实例详情接口」得到<br>**示例值**：["123456789"]<br>**数据校验规则**：<br>- 长度范围：`1` ～ `10`
comment | string | 否 | 评论，请控制在500个字符内<br>**示例值**："同意"

### 请求体示例
```json
{
    "instance_code": "81D31358-93AF-92D6-7425-01A5D67C4E71",
    "task_ids": [
        "123456789"
    ],
    "comment": "同意"
}
```

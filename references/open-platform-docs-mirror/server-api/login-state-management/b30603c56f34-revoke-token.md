# 撤销 user_access_token 或 refresh_token
OAuth 令牌接口，可用于撤销 <code>user_access_token</code> 或 <code>refresh_token</code>。

- 用于主动撤销 `user_access_token` 或 `refresh_token`，适用于用户登出、应用侧清理登录态等场景。
- 为避免泄露凭证状态，无效、过期、已撤销或格式非法的 token 仍按成功处理。具体参数说明见下文请求体。
**注意事项**：本接口实现遵循 [RFC 7009 - OAuth 2.0 Token Revocation](https://datatracker.ietf.org/doc/html/rfc7009)，建议优先使用标准 OAuth 客户端库进行接入。

基本 | &nbsp;
---|---
HTTP URL | https://accounts.feishu.cn/oauth/v1/revoke
HTTP Method | POST
支持的应用类型 | Custom App、Store App
权限要求<br>**调用该 API 所需的权限。开启其中任意一项权限即可调用<br>** | 无

### 客户端认证

本接口需要进行客户端认证，认证方式与 OAuth 2.0 标准撤销端点保持一致（二选一）：

- 方式一：使用 `HTTP Basic Authorization`（推荐）。
- 方式二：在请求体中传递 `client_id` 和 `client_secret`。

### 请求头

名称 | 类型 | 必填 | 描述
---|---|---|---
Content-Type | string | 是 | 请求体类型。<br>**固定值：**`application/x-www-form-urlencoded`

### 请求体

名称 | 类型 | 必填 | 描述
---|---|---|---
token | string | 是 | 需要撤销的 token，支持传入 `user_access_token` 或 `refresh_token`。<br>- 当 `token` 参数传入 `user_access_token` 时，仅当前 `user_access_token` 失效；若对应的 `refresh_token` 仍有效，可继续通过[刷新 user_access_token](https://open.feishu.cn/document/uAjLw4CM/ukTMukTMukTM/authentication-management/access-token/refresh-user-access-token)获取新的 `user_access_token`。<br>- 当 `token` 参数传入 `refresh_token` 时，会使当前授权关系下的刷新能力失效。此时如需重新获取访问凭证，请重新走[获取 user_access_token](https://open.feishu.cn/document/uAjLw4CM/ukTMukTMukTM/authentication-management/access-token/get-user-access-token)流程。<br>**示例值：**`eyJhbGciOiJFUzI1NiIs**********XXOYOZz1mfgIYHwM8ZJA`
client_id | string | 否 | 应用的 App ID。仅当你选择在请求体中传递客户端凭证，而不是使用 `HTTP Basic Authorization` 时需要提供。应用凭证 App ID 和 App Secret 获取方式：<br>1. 登录[飞书开发者后台](https://open.feishu.cn/app)。<br>2. 进入应用详情页，在左侧导航栏，单击 **凭证与基础信息**。<br>3. 在 **应用凭证** 区域，获取并保存 **App ID** 和 **App Secret**。<br>**示例值：**`cli_a5ca35a685b0x26e`
client_secret | string | 否 | 应用的 App Secret。仅当你选择在请求体中传递客户端凭证，而不是使用 `HTTP Basic Authorization` 时需要提供。应用凭证 App ID 和 App Secret 获取方式：<br>1. 登录[飞书开发者后台](https://open.feishu.cn/app)。<br>2. 进入应用详情页，在左侧导航栏，单击 **凭证与基础信息**。<br>3. 在 **应用凭证** 区域，获取并保存 **App ID** 和 **App Secret**。<br>**示例值：**`baBqE5um9LbF*************1sqpXlwy`
token_type_hint | string | 否 | token 类型提示参数，用于帮助服务端更快定位 token 类型。<br>仅用于提示 `token` 参数传入的是 `user_access_token` 还是 `refresh_token`；服务端会基于 `token` 参数的实际内容自动识别类型。<br>**可选值：**<br>- `access_token`：表示传入的 `token` 预期为 `user_access_token`<br>- `refresh_token`：表示传入的 `token` 预期为 `refresh_token`<br>**注意：**<br>- 当前参数仅作提示使用，不会影响服务端最终采用的撤销维度。<br>- 如果不传该参数，服务端会自动识别 token 类型。<br>- 如果传入除 `access_token`、`refresh_token` 外的其他值，接口会报错。<br>**示例值：**`refresh_token`

### 请求体示例
使用 HTTP Basic Authorization：

```http
POST https://accounts.feishu.cn/oauth/v1/revoke
Authorization: Basic Y2xpX2E1Y2EzNWE2ODViMHgyNmU6YmFCcUU1dW05TGJG***MASKED***c3FQWGx3eQ==
Content-Type: application/x-www-form-urlencoded

token=[REDACTED]**********XXOYOZz1mfgIYHwM8ZJA&token_type_hint=refresh_token
```

直接在请求体中传递应用凭证信息：

```http
POST https://accounts.feishu.cn/oauth/v1/revoke
Content-Type: application/x-www-form-urlencoded

token=[REDACTED]**********XXOYOZz1mfgIYHwM8ZJA&client_id=cli_a5ca35a685b0x26e&client_secret=baBqE5um9LbF*************1sqpXlwy&token_type_hint=refresh_token
```

## 响应

调用成功时返回 HTTP 200，且无响应体（empty response body）。

调用失败时，响应体类型为 `application/json; charset=utf-8`。

### 响应体

名称 | 类型 | 描述
---|---|---
code | int | 错误码，仅在请求失败时返回
error | string | 错误类型，仅在请求失败时返回
error_description | string | 具体错误信息，仅在请求失败时返回

### 响应体示例

成功响应示例：

```http
HTTP/1.1 200 OK
```

说明：成功时仅返回 HTTP 状态码，不返回响应体内容。

失败响应示例：
```json
{
    "code": 20001,
    "error": "invalid_request",
    "error_description": "The token_type_hint parameter should be access_token or refresh_token."
}
```

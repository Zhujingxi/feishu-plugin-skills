# 搜索群组

用户可以通过关键字搜索可见群组，可见性和套件内搜索一致。
**注意事项**：该接口是 https://open.feishu.cn/document/server-docs/group/chat/search 搜群组能力的升级版，推荐使用该接口

## 请求

基本 | &nbsp;
---|---
HTTP URL | https://open.feishu.cn/open-apis/im/v2/chats/search
HTTP Method | POST
接口频率限制 | [100 次/分钟](https://open.feishu.cn/document/ukTMukTMukTM/uUzN04SN3QjL1cDN)
支持的应用类型 | Custom App、Store App
权限要求<br>**调用该 API 所需的权限。开启其中任意一项权限即可调用**<br>开启任一权限即可 | 获取与更新群组信息(im:chat)<br>查看群信息(im:chat:read)<br>获取群组信息(im:chat:readonly)
字段权限要求 | **注意事项**：该接口返回体中存在下列敏感字段，仅当开启对应的权限后才会返回；如果无需获取这些字段，则不建议申请<br>获取用户 user ID(contact:user.employee_id:readonly)

### 请求头

名称 | 类型 | 必填 | 描述
---|---|---|---
Authorization | string | 是 | `tenant_access_token`<br>或<br>`user_access_token`<br>**值格式**："Bearer `access_token`"<br>**示例值**："Bearer u-7f1bcd13fc57d46bac21793a18e560"<br>[了解更多：如何选择与获取 access token](https://open.feishu.cn/document/uAjLw4CM/ugTN1YjL4UTN24CO1UjN/trouble-shooting/how-to-choose-which-type-of-token-to-use)
Content-Type | string | 是 | **固定值**："application/json; charset=utf-8"

### 查询参数

名称 | 类型 | 必填 | 描述
---|---|---|---
page_size | int | 否 | **示例值**：10
page_token | string | 否 | 分页标记，第一次请求不填，表示从头开始遍历；分页查询结果还有更多项时会同时返回新的 page_token，下次遍历可采用该 page_token 获取查询结果<br>**示例值**：eVQrYzJBNDNONlk4VFZBZVlSdzlKdFJ4bVVHVExENDNKVHoxaVdiVnViQT0=
user_id_type | string | 否 | 用户 ID 类型<br>**示例值**：open_id<br>**可选值有**：<br>- open_id：标识一个用户在某个应用中的身份。同一个用户在不同应用中的 Open ID 不同。[了解更多：如何获取 Open ID](https://open.feishu.cn/document/uAjLw4CM/ugTN1YjL4UTN24CO1UjN/trouble-shooting/how-to-obtain-openid)<br>- union_id：标识一个用户在某个应用开发商下的身份。同一用户在同一开发商下的应用中的 Union ID 是相同的，在不同开发商下的应用中的 Union ID 是不同的。通过 Union ID，应用开发商可以把同个用户在多个应用中的身份关联起来。[了解更多：如何获取 Union ID？](https://open.feishu.cn/document/uAjLw4CM/ugTN1YjL4UTN24CO1UjN/trouble-shooting/how-to-obtain-union-id)<br>- user_id：标识一个用户在某个租户内的身份。同一个用户在租户 A 和租户 B 内的 User ID 是不同的。在同一个租户内，一个用户的 User ID 在所有应用（包括商店应用）中都保持一致。User ID 主要用于在不同的应用间打通用户数据。[了解更多：如何获取 User ID？](https://open.feishu.cn/document/uAjLw4CM/ugTN1YjL4UTN24CO1UjN/trouble-shooting/how-to-obtain-user-id)<br>**默认值**：`open_id`<br>**当值为 `user_id`，字段权限要求**：<br>获取用户 user ID(contact:user.employee_id:readonly)

### 请求体

名称 | 类型 | 必填 | 描述
---|---|---|---
query | string | 否 | query (长度范围：0 ～ 50 字符)<br>**示例值**："部门群"
filter | chat_search_filter | 否 | filter
search_types | string\[\] | 否 | 群组类型<br>**示例值**：["private"]<br>**可选值有**：<br>- private：私有群<br>- external：外部群<br>- public_joined：已加入的公开群 <br>- public_not_joined：未加入的公开群<br>**数据校验规则**：<br>- 长度范围：`0` ～ `4`
member_ids | string\[\] | 否 | 群成员ID<br>成员ID即是User ID，获取方式：https://open.feishu.cn/document/server-docs/contact-v3/user/get?appId=cli_a9228da536381cc0<br>**示例值**：["ou-7890123456abcdef"]<br>**数据校验规则**：<br>- 长度范围：`0` ～ `500`
is_manager | boolean | 否 | 是否自己创建或者管理的群组<br>**示例值**：true
disable_search_by_user | boolean | 否 | 是否关闭以人搜群功能： 先通过群成员名搜索，再搜群组(默认开启)<br>**示例值**：true
chat_modes | string\[\] | 否 | 群模式筛选器，支持按普通群/话题群过滤<br>**示例值**：["thread"]<br>**可选值有**：<br>- default：普通群<br>- thread：话题群<br>**数据校验规则**：<br>- 长度范围：`0` ～ `2`
sorter | string | 否 | sorter<br>**示例值**："create_time_desc"<br>**可选值有**：<br>- create_time_desc：按照群组创建时间降序排序<br>- update_time_desc：按照群组更新时间降序排序<br>- member_count_desc：按照群组成员数量降序排序

### 请求体示例
```json
{
    "query": "部门群",
    "filter": {
        "search_types": [
            "private"
        ],
        "member_ids": [
            "ou-7890123456abcdef"
        ],
        "is_manager": true,
        "disable_search_by_user": true,
        "chat_modes": [
            "thread"
        ]
    },
    "sorter": "create_time_desc"
}
```

## 响应

### 响应体

名称 | 类型 | 描述
---|---|---
code | int | 错误码，非 0 表示失败
msg | string | 错误描述
data | \- | \-

id | string | 群组ID

meta_data | chat_search_meta | 群组元信息
chat_id | string | 群组 ID
create_time | string | 创建时间(iso8601)
update_time | string | 更新时间(iso8601)
external | boolean | 是否是外部群
chat_mode | string | 群模式，group：群组 topic: 话题
description | string | 群描述
avatar | string | 群头像URL
name | string | 群名称
owner_id | string | 群主ID
owner_id_type | string | 群主ID类型
tenant_key | string | tenant key
chat_status | string | 群状态<br>**可选值有**：<br>- normal：正常<br>- dissolved：解散<br>- dissolved_save：解散并保留
total | int | total
has_more | boolean | 是否还有更多项
page_token | string | 分页标记，当 has_more 为 true 时，会同时返回新的 page_token，否则不返回 page_token
notice | string | 搜索补充提示信息，返回本次搜索的额外说明，例如：query被截断；搜索结果不全等
